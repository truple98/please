
도시밖 지방 체험 – 와이어프레임 설계서 (텍스트 기반 레이아웃)

📝 작성 목적
이 문서는 MVP 수준의 화면 구성을 시각적 레이아웃 구조로 정의하여
프론트 개발자(Cursor), 디자이너, 협업자들이 각 컴포넌트 위치와 기능을 이해할 수 있도록 지원합니다.

────────────────────────────────────────
📄 [1] 홈 페이지 (/)
────────────────────────────────────────
[상단]
- 로고 / 서비스명
- [시작하기] 버튼 → /chat 연결
- 상단 메뉴: 소개 | 마이페이지

[중단]
- 서비스 소개 (텍스트 + 일러스트)
- “어떤 경험을 찾고 계신가요?” 문구
- AI 추천 시작 버튼 (중앙 배치)

[하단]
- 추천 체험 미리보기 (썸네일 카드 3개)
- 푸터: 개인정보처리방침 / 문의

────────────────────────────────────────
📄 [2] 챗봇 대화 페이지 (/chat)
────────────────────────────────────────
[상단 고정]
- 로고 + “체험 큐레이터 Emoti와 대화 중입니다”

[중단 영역]
- 메시지 뷰 (좌측: 사용자 / 우측: 챗봇 말풍선)
- 자동 입력 선택지 (버튼형 답변들)

[하단]
- 사용자 직접 입력창 (선택)
- [다음 질문] 버튼 또는 자동 진행

────────────────────────────────────────
📄 [3] 체험 추천 결과 페이지 (/recommend)
────────────────────────────────────────
[상단]
- “당신에게 어울리는 체험을 찾아봤어요”

[중단]
- 체험 카드 리스트 (그리드 2열)
  ┌─────────────┐
  │ 이미지       │
  │ 제목         │
  │ 지역 / 한줄소개 │
  │ [찜하기] [자세히보기] │
  └─────────────┘

[하단]
- “더 많은 추천 보기” 버튼
- “챗봇으로 돌아가기” 버튼

────────────────────────────────────────
📄 [4] 체험 상세 페이지 (/experience/:id)
────────────────────────────────────────
[상단]
- 체험명 / 지역명 / 이미지 슬라이드

[중단]
- 체험 설명
- 시간표 / 가격 / 위치 지도
- 후기 평균 별점 + 후기 미리보기

[하단]
- [찜하기] 버튼
- [외부 예약 링크] 버튼
- [후기 전체 보기] 링크

────────────────────────────────────────
📄 [5] 마이페이지 (/mypage)
────────────────────────────────────────
[탭 구성]
- 🧡 찜한 체험
- 📅 예약 내역
- ✍ 후기 작성 내역

[각 탭 콘텐츠]
- 카드 리스트 UI (체험 썸네일 + 액션 버튼)

────────────────────────────────────────
📄 [6] 후기 작성 페이지 (/review)
────────────────────────────────────────
[상단]
- “체험은 어떠셨나요?” 문구

[중단]
- 별점 선택 (1~5)
- 후기 텍스트 입력 필드
- 이미지 업로드 (선택)

[하단]
- [후기 제출하기] 버튼

────────────────────────────────────────

📌 기타 공통 UI 요소
- 반응형 (모바일 우선)
- 상단 네비게이션 고정
- 챗봇 말풍선은 타이핑 효과 + 일시 정지 가능
- 다크모드 지원 (선택)

※ 이 구조는 MVP 기준이며, 지도 기반 탐색이나 관리자 기능은 추후 확장 예정
