
도시밖 지방 체험 – 정보 구조도 (Information Architecture)

1. 전체 구조 개요
서비스의 흐름은 사용자의 "탐색 → 추천 대화 → 체험 선택 → 피드백" 여정을 중심으로 구성됩니다.
초기 MVP 단계에서는 다음과 같은 핵심 메뉴와 페이지가 포함됩니다.

2. 주요 메뉴 및 페이지 트리 구조

- [홈]
  - 서비스 소개 (텍스트 + 일러스트 또는 스크롤형 구조)
  - [→ 챗봇 추천 시작 버튼]

- [AI 체험 추천 챗봇]
  - 사용자 정보 입력 (챗봇 대화형)
    - 성향 질문 5~7개
    - 관심사, 거주지, 이동 가능 거리 등
  - 추천 결과 페이지
    - 카드 형태의 추천 체험 리스트
    - [체험 상세 보기], [찜하기], [외부 예약 연결]

- [체험 상세 페이지]
  - 체험 개요 (사진, 한 줄 소개, 위치)
  - 제공 내용 / 시간표
  - 후기 요약
  - 지도 보기
  - [찜하기], [예약하기], [다른 체험 추천받기]

- [마이페이지]
  - 찜한 체험 보기
  - 예약 내역 보기
  - 후기 작성 내역 및 작성하기
  - 나의 성향 정보 확인 및 수정

- [후기 작성 페이지]
  - 별점 평가
  - 자유 입력 텍스트
  - 이미지 첨부 가능 (선택)

- [관리자 페이지] (초기 MVP에는 간략하게 구성)
  - 체험 등록
  - 체험 수정
  - 후기 관리 (신고/비노출 처리)

3. 사용자 흐름 기반 UX 구조

[홈] → [챗봇 대화 시작]
    → [질문 응답]
        → [추천 결과 리스트]
            → [체험 상세 보기]
                → [찜하기] or [예약하기]
                    → [마이페이지] → [후기 작성]

4. 보조 기능 및 UI 요소

- 상단 고정 메뉴 (홈 / 추천 / 마이페이지)
- 모바일 우선 반응형 UI
- 다크모드 대응 (선택)
- 챗봇 대화 기록 저장 여부 (ON/OFF 선택 가능)

5. 추후 확장 고려 요소

- 체험 지역 기반 지도 인터페이스
- 지역별 인기 체험 랭킹
- 사용자 맞춤 대시보드 (선호 카테고리 기반 추천)
