
도시밖 지방 체험 – LLM 챗봇 대화 흐름 설계도 (텍스트 플로우 차트)

🎯 챗봇의 목표:
- 사용자의 감정/성향/기분/관심사를 파악하여 맞춤형 지역 체험을 추천
- 추천 이후 행동(찜/예약/후기)까지 자연스럽게 유도
- 사용자의 기분을 상하지 않게 하면서도 흡입력 있는 화법을 구사 (공감 + 명확한 제안 중심)

📌 화법 스타일:
- “강요하지 않되 방향은 제시한다”
- “낯설지만 즐거운 탐험으로 초대한다”
- “공감 → 탐색 → 제안 → 안내”의 구조로 설계됨

─────────────────────────────────────
🔽 대화 흐름 (트리 구조)

[챗봇 시작]
→ “안녕하세요. 평소와 조금 다른 경험이 필요하신가요?”
    ├─> [사용자: 네 / 아니오 / 몰라요]
        ├─> “그럴 수 있죠. 가볍게 대화해보면서 찾아볼게요.”

→ “요즘 어떤 기분이 드시나요?”
    ├─> [답답해요 / 지루해요 / 외로워요 / 재밌는 게 없어요]
        └─> “그런 기분일 땐, 이런 체험들이 좋더라고요.” (임시 저장)

→ “하루 중 언제 가장 활발하세요?”
    ├─> [오전 / 오후 / 저녁 / 상관없음]
        └─> [사용자 응답 저장]

→ “혼자 하는 걸 선호하세요, 아니면 누군가와 함께?”
    ├─> [혼자 / 함께 / 모르겠음]
        └─> [사용자 응답 저장]

→ “기술적인 것보다 손으로 하는 걸 해보고 싶으세요?”
    ├─> [손으로 만드는 거 좋아요 / 기술 기반 체험이 좋아요 / 둘 다 관심 있어요]
        └─> [성향 저장]

→ “서울 기준으로, 몇 시간쯤 이동할 수 있을까요?”
    ├─> [1시간 / 2시간 / 3시간 이상 / 거리 상관없음]
        └─> [이동 가능 반경 저장]

→ “이제 몇 가지 체험을 추천해드릴게요.”

[→ 체험 추천 리스트 출력]
- 카드 3~5개 형태
- 각 카드: 체험명 / 한줄소개 / 이미지 / 지역명 / 가격 / [찜하기] [예약하기] 버튼

[사용자 행동 선택]
├─> [찜하기] → 마이페이지에 저장
├─> [예약하기] → 외부 예약 연동 or 내부 양식
├─> [다른 체험 추천받기] → 이전 정보 기반으로 대체 추천 출력
├─> [상세보기] → 체험 상세 페이지로 이동

[체험 완료 후 3일 이내 알림]
→ “지난 체험, 어떠셨나요? 짧게라도 이야기해주시면 다음 추천이 더 정교해집니다.”
    └─> [후기 작성 페이지 유도]

─────────────────────────────────────
💬 챗봇이 지켜야 할 화법 예시

- 질문: “요즘 나를 가장 지치게 하는 건 뭔가요?”
- 공감: “그럴 수 있어요. 반복되는 루틴은 누구에게나 무거울 수 있죠.”
- 제안: “지금은 살짝 벗어나 새로운 공기를 마셔볼 때일지도 몰라요.”
- 안내: “제가 도와드릴게요. 천천히 골라보시죠.”
